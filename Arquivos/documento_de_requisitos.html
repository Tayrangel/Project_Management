Introdução

Este documento descreve os requisitos do Sistema de Gerenciamento de Documentos Digitais a ser desenvolvido pela equipe da Techwise. O sistema tem como objetivo principal gerenciar documentos de forma eletrônica, permitindo sua criação, armazenamento, compartilhamento, busca e recuperação. O sistema será composto por um banco de dados PostgreSQL, um backend Django e um frontend React.

  • Requisitos Funcionais

  • Cadastro de Usuários

    O sistema deve permitir o cadastro de usuários com os seguintes campos: nome, e-mail, senha e perfil de acesso (administrador ou usuário comum).

  • Login de Usuários

    O sistema deve permitir o login de usuários cadastrados através de e-mail e senha.

  • Criação de Documentos

    O sistema deve permitir a criação de documentos com os seguintes campos: título, descrição, categoria, data de criação e upload de arquivo.

  • Armazenamento de Documentos

    O sistema deve armazenar os documentos criados pelos usuários em um banco de dados PostgreSQL.

  • Compartilhamento de Documentos

    O sistema deve permitir que usuários com permissão possam compartilhar documentos com outros usuários do sistema.

  • Busca e Recuperação de Documentos

    O sistema deve permitir a busca e recuperação de documentos através de filtros como título, categoria, data de criação e descrição.

  • Controle de Versão de Documentos

    O sistema deve permitir o controle de versão dos documentos criados, mantendo um histórico das alterações realizadas.

  • Autenticação e Autorização

    O sistema deve possuir um sistema de autenticação e autorização para garantir a segurança dos documentos e das informações dos usuários.

  • Requisitos Não Funcionais

  • Performance

    O sistema deve ser capaz de suportar um grande volume de documentos e usuários simultâneos, sem comprometer a performance.

  • Usabilidade

    O sistema deve possuir uma interface amigável e de fácil uso para todos os usuários, mesmo aqueles sem conhecimento técnico.

  • Segurança

    O sistema deve ser seguro, garantindo a proteção dos documentos e informações dos usuários, além de evitar acessos não autorizados.

  • Disponibilidade

    O sistema deve estar disponível 24 horas por dia, 7 dias por semana, com exceção de manutenções programadas.

  • Requisitos de Integração

  • Controle de versão

    O sistema deve utilizar o Git e o GitHub para controle de versão do código fonte do projeto.

  • Banco de Dados

    O sistema deve utilizar o PostgreSQL como banco de dados.

  • Backend e Frontend

    O sistema deve utilizar o Django como backend e o React como frontend.
